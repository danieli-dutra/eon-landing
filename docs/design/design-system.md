# EON | Design System


# O que é o Design System da EON?

O Design System da EON reúne os princípios que orientam todas as decisões visuais da experiência.

Ele não existe apenas para padronizar interfaces.

Seu principal objetivo é preservar a identidade da EON à medida que o projeto evolui.

Mais do que definir cores, tipografia ou componentes, este documento registra a lógica por trás dessas escolhas.

Sempre que uma nova interface, seção ou componente for criado, ele deve parecer uma continuação natural daquilo que já existe.

---

# Filosofia

Toda decisão visual deve reforçar a narrativa da EON.

O design não procura chamar atenção para si mesmo.

Ele cria as condições para que a mensagem seja percebida com clareza.

Nada existe apenas por estética.

Cada escolha deve possuir uma intenção.

A interface deve transmitir calma, confiança e permanência.

---

# Princípios Fundamentais

O sistema visual da EON é orientado pelos seguintes princípios.

## Clareza antes de complexidade

- Interfaces devem eliminar ruído.

- Cada elemento precisa justificar sua existência.

- Quando houver dúvida entre adicionar ou remover algo, a simplicidade deve prevalecer.

---

## Conteúdo antes de decoração

- A interface nunca compete com a mensagem.

- Recursos visuais existem para apoiar a narrativa, nunca para substituí-la.

---

## Movimento implícito antes de movimento explícito

- A sensação de movimento nasce primeiro da composição visual.

- Linhas, formas, espaçamentos e alinhamentos conduzem naturalmente o olhar.

- As animações apenas reforçam essa percepção.

---

## Geometria antes de ilustração

- A linguagem visual da EON utiliza formas simples, proporcionais e consistentes.

- Sempre que possível, símbolos e elementos gráficos devem nascer de estruturas geométricas, evitando representações literais.

---

## Consistência antes de novidade

- Novos elementos devem ampliar o sistema existente, nunca criar uma linguagem paralela.

- O objetivo não é surpreender em cada tela, mas construir reconhecimento ao longo do tempo.

---

# Hierarquia

Toda interface deve estabelecer uma leitura clara.

A prioridade visual segue sempre a mesma lógica:

1. Mensagem principal
2. Conteúdo de apoio
3. Ações disponíveis
4. Informações complementares

O visitante nunca deve precisar decidir por onde começar.

A hierarquia deve tornar essa escolha natural.

---

# Ritmo

A experiência da EON alterna momentos de intensidade e contemplação.

Grandes títulos convivem com amplos espaços vazios.

Seções densas são seguidas por pausas visuais.

Esse contraste cria uma navegação fluida e evita fadiga durante a leitura.

O ritmo faz parte da narrativa.

---

# Espaço Negativo

O espaço vazio é um elemento de composição.

Ele organiza a informação, direciona a atenção e define o ritmo da interface.

Nenhuma seção deve parecer apertada ou excessivamente preenchida.

Sempre que possível, o espaço negativo deve comunicar tanto quanto os elementos visíveis.

---

# Contraste

O contraste é utilizado para estabelecer prioridade, não para criar impacto gratuito.

A interface trabalha principalmente através de:

- contraste entre claro e escuro;
- contraste de escala tipográfica;
- contraste de peso visual;
- contraste entre silêncio e informação.

Elementos de destaque devem existir em quantidade limitada para preservar sua força.

---

# Tipografia

A tipografia representa dois papéis distintos.

## Clash Display

Utilizada exclusivamente para títulos e elementos editoriais.

Sua personalidade transmite presença, identidade e permanência.

Ela conduz o olhar e define o tom da comunicação.

---

## Inter Tight

Utilizada em textos de apoio, navegação e interface.

Foi escolhida pela alta legibilidade em diferentes tamanhos e dispositivos.

Enquanto a Clash estabelece personalidade, a Inter garante conforto durante a leitura.

As duas famílias tipográficas foram escolhidas para atuar de forma complementar.

---

# Cor

A paleta da EON é intencionalmente reduzida.

Cada cor possui uma função específica.

## Preto

Representa profundidade e cria o ambiente onde a narrativa acontece.

---

## Branco

Prioriza legibilidade e estabelece contraste.

---

## Dourado

Representa significado.

Por isso é utilizado com moderação, destacando apenas elementos que possuem importância narrativa.

Seu uso excessivo reduz seu impacto.

---

## Azul

É utilizado como cor de apoio.

Sua função é equilibrar a composição sem competir com o dourado.

---

# Grid

Toda composição deve partir de uma estrutura consistente.

O grid organiza alinhamentos, espaçamentos e proporções entre elementos.

Mesmo quando não é percebido pelo visitante, ele garante equilíbrio visual e continuidade entre diferentes seções.

---

# Componentes

Os componentes da EON seguem os mesmos princípios presentes em toda a interface.

Eles devem ser:

- simples;
- previsíveis;
- consistentes;
- discretos.

O componente nunca é o protagonista.

Ele existe para apoiar a experiência.

Sempre que um componente exigir atenção maior do que o conteúdo que apresenta, sua implementação deve ser reconsiderada.

---

# Consistência

A consistência é o resultado esperado de todas as decisões anteriores.

Ela não significa repetição.

Significa que elementos diferentes compartilham a mesma lógica de construção.

Uma nova página, componente ou interação deve parecer pertencer à EON mesmo antes da presença da logomarca.

Esse é o principal objetivo do Design System.

---

# Princípio Final

O Design System da EON não procura responder apenas como desenhar uma interface.

Ele procura responder como tomar decisões que preservem a identidade da experiência.

Quando uma nova solução parecer naturalmente parte da EON, mesmo sem seguir um modelo existente, este sistema estará cumprindo seu papel.
